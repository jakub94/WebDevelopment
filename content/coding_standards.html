Our coding standards - or what's left of them

We can't remember too clearly, but at some point in time we all learned the rules and standards needed to build a website that will pass a validation and can be accessed by everyone - sober and drunken people.

Sadly, the standards change. And languages change. Fast. How are we supposed to keep up with all that when we're so busy brewing our favorite Vodka? See. That's why we keep a handy list of links here. Mainly for us. But maybe you're interested too. Whatever.

HTML & Semantics

  • Code Academy

    Can't even remember the basics? Back to the Academy then! Here you can (re-)learn the fundamentals of web development.

  • W3C: HTML Element Reference

    Man, how many of those elements are there? I don't know. You don't know. But this reference knows. Cool!

  • Sitepoint: HTML Reference

    W3C is fine and all, but they certainly lack sexyness. Well, no worries, Sitepoint to the rescue! And they also have an Attribute Reference.

  • Webplatform: HTML Reference

    I don't know too much, but what I do know is: one can never have enough references. So take this. Elements, Attributes and what-not!
